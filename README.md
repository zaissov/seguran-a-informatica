# seguran-a-informatica




Plano Hardcore de Estudo em Cybersecurity – 6 Meses

Este repositório acompanha meu progresso no estudo intensivo de cibersegurança e testes de invasão.

Objetivo

Preparar para trabalho remoto em cibersegurança e testes de invasão, com estudo teórico + prática diária, usando MacBook e laboratório virtual.

Livros Base
	•	Segurança de Computadores e Teste de Invasão
	•	Hacking: The Art of Exploitation
	•	The Web Application Hacker’s Handbook
	•	The Mobile Application Hacker’s Handbook (opcional)
	•	Clean Code (boas práticas)
	•	The Linux Command Line (linha de comando avançada)

Ferramentas Necessárias
	•	Kali Linux VM, VirtualBox / VMware
	•	Wireshark, tcpdump, Nmap, Recon-ng, Maltego, Burp Suite, OWASP ZAP, Metasploit, John the Ripper, THC Hydra
	•	Sysinternals, ELK Stack, OpenSSL, GDB, Snapshots VM

Estrutura do Repositório
	•	/semanas → Checklists semanais em Markdown
	•	/projetos → Projetos práticos por semana
	•	/logs → Capturas de telas, logs, relatórios
	•	/cheatsheets → Comandos principais e dicas rápidas

Checklist Semanal

Cada semana possui tarefas de estudo, prática e documentação:

Semana 1–2: Fundamentos & Ética
	•	Instalar Kali Linux VM
	•	Mapear rede local do MacBook
	•	Criar regras de engajamento em Markdown
	•	Simular ataque ético em rede fictícia
	•	Criar diagrama de perfis de hackers
	•	Exercícios de revisão
	•	Projeto prático resumido em Markdown

Semana 3–4: Reconhecimento & Engenharia Social
	•	Mapear rede e IPs locais
	•	Criar cenário de phishing seguro
	•	Coletar informações de perfis fictícios nas redes sociais
	•	Enumeração de rede (DNS/IP scan)
	•	Documentar prevenção de engenharia social
	•	Projeto prático resumido em Markdown

Semana 5–6: Escaneamento & Sniffers
	•	TCP/UDP scans
	•	Captura de pacotes com Wireshark
	•	Identificação de protocolos
	•	Detectar sniffers na rede
	•	Criar relatório em Markdown

Semana 7–8: Vulnerabilidades TCP/IP
	•	IP spoofing em VM
	•	SYN flood teste seguro
	•	ICMP attacks teste seguro
	•	Configuração de firewall local
	•	Criação de túnel IPSec
	•	Relatório de vulnerabilidades

Semana 9–10: Criptografia & Senhas
	•	Criar chaves simétricas/assimétricas
	•	Testar ataques a hashes MD5/SHA1
	•	Ataque dicionário e força bruta
	•	Simular keyloggers em VM
	•	Relatório em Markdown

Semana 11–12: Aplicações Web & OWASP
	•	Configurar DVWA
	•	SQL Injection
	•	XSS e CSRF
	•	Teste de autenticação fraca
	•	Documentação de vulnerabilidades

Semana 13–14: Malware e Exploits
	•	Criar backdoor seguro em VM
	•	Buffer overflow básico e avançado em C/C++
	•	Testar exploits de linguagem
	•	Simular malware em VM segura
	•	Criar relatório de exploração e defesa

Semana 15–16: Sistemas Operacionais
	•	Testar contas padrão e senhas no Windows/Linux
	•	Elevação de privilégios
	•	Testar vulnerabilidades do kernel
	•	Auditar logs e permissões
	•	Exploração e defesa documentada

Semana 17–18: Tratamento de Incidentes
	•	Mapear possíveis incidentes
	•	Simular logs e alertas
	•	Testar contenção e isolamento de VM
	•	Erradicar vulnerabilidades
	•	Recuperação de sistemas
	•	Criar plano formal de resposta

Semana 19–20: Projeto Final Integrado
	•	Criar rede simulada (Windows + Linux + DVWA)
	•	Testar todas as etapas de pentest
	•	Gerar relatório completo profissional em Markdown

Dicas GitHub
	•	Criar uma branch por semana ou módulo
	•	Marcar tarefas concluídas com [x]
	•	Commit diário com notas, resultados e screenshots
	•	Atualizar /logs com evidências práticas
	•	Manter README atualizado com progresso
	•	Criar cheatsheets para cada ferramenta em /cheatsheets
